# Phase 1 — 체크리스트 시트 (`체크리스트`)

## 열 배치 (6열 고정)
```
A=단계 | B=항목 | C=계획서 기재값 | D=검증/기준값 | E=판정 | F=근거 위치
```
E열 판정: `✅` / `⚠️` / `❌` 기호만 (추가 텍스트 절대 금지)

> ⚠️/❌ 로 판정된 항목과 미기재·확인불가 항목은 `업체확인필요사항` 시트에도 누적 기록한다.

---

## 행 구조
```
행1 : 제목 (A1:F1 병합)
행2 : 대상 현장
행4 : 요약 레이블
행5 : 요약 수식 (COUNTA/COUNTIF)
행6 : 열 헤더
행7~행27 : 필수 21개 항목 (순서·텍스트 고정)
행28~  : 추가 발견사항
```

요약 수식:
```python
ws["C4"]="총항목"; ws["D4"]="✅"; ws["E4"]="⚠️"; ws["F4"]="❌"
ws["C5"]="=COUNTA(B7:B300)"
ws["D5"]='=COUNTIF(E7:E300,"✅")'
ws["E5"]='=COUNTIF(E7:E300,"⚠️")'
ws["F5"]='=COUNTIF(E7:E300,"❌")'
```

## 필수 21개 항목 — 7~27행 (A/B열 텍스트 고정)

| 행 | A열(단계) | B열(항목) — 이 텍스트 그대로 |
|----|-----------|-------------------------------|
| 7  | 1단계 | 부하계산서 첨부 여부 |
| 8  | 1단계 | 부하계산서 건물명 일치 여부 |
| 9  | 1단계 | 부하계산서 계산방법 명시 여부 |
| 10 | 2단계 | 냉방 시간최대부하 일치 여부 (계산서 vs 기재) |
| 11 | 2단계 | 냉방 최대부하일 부하량 일치 여부 |
| 12 | 2단계 | 연면적 m²→평 환산 검증 (÷3.30579) |
| 13 | 2단계 | 냉난방면적 m²→평 환산 검증 (÷3.30579) |
| 14 | 3단계 | 운전계획 심야시간 기준 (22시 시작 여부) |
| 15 | 3단계 | 심야시간대 냉방부하 유무 확인 |
| 16 | 3단계 | 심야 최대냉방부하(RT) ≤ 기타시간대 최대냉방부하(RT) × 60% |
| 17 | 3단계 | 심야 냉방부하량(RTh) ≤ 일일 총냉방부하량(RTh) × 40% |
| 18 | 4단계 | 축냉설비 심야전력 입력용량 ≥ 20kW |
| 19 | 4단계 | 축열률 ≥ 40% |
| 20 | 4단계 | 상시전력 계약종별 확인 (산업용전력(을) 아님 여부) |
| 21 | 4단계 | 열원기기 입력 합계 vs 기재값 (±1kW) |
| 22 | 4단계 | 열원기기 용량 부족 여부 (모드별 용량 vs 시간대별 운전량) |
| 23 | 4단계 | 열원기기 과다 산정 여부 (모드별 용량 ≥ 운전량 × 120%) |
| 24 | 4단계 | 축열조 설계용량 vs 운전계획 최대누계 |
| 25 | 4단계 | 축열조 규격으로 계산한 축열능력 vs 설계값 검증 |
| 26 | 5단계 | (냉온열겸용) 난방 총부하 ≤ 냉방 총부하 |
| 27 | 5단계 | 펌프 순환유량 LPM/RT 적정성 (기준 10, ±10% = 9~11) |

> **행28~ 추가 발견사항은 `decision_rules.md` §5 교차검증 카탈로그에서 발견된 항목만**,
> 카탈로그 순서대로 기록한다 (개방형 추가 금지). A열=`추가`, B열=카탈로그 표준 제목, E열=`⚠️`.
> C·D열 고정 내용은 아래 **시트1 셀 맵**을 그대로 사용한다.
> F열은 `근거 위치`다(v17) — 전 행에 원문 파일명·페이지를 적는다.
> 섹션 헤더 행(▶, ■) 금지 / A열에 번호(1,2,3) 금지.

## 시트1 셀 맵 (`체크리스트`)


- B2 = `{수용가명} — 공급업체: {공급업체명} / 접수지점: {접수지점}` (이 형식 그대로, 다른 정보 추가 금지)
  - `{수용가명}` = **설치내역서 표지 '수용가명' 필드 원문 그대로** (고객 상호나 주소를 넣지 않는다).
    (예: `○○타워 노후 냉방설비 교체 공사계약` — 문서마다 다르므로 표지 원문을 그대로 옮긴다)
- 행4·5 요약, 행6 헤더, 행7~27 필수 21개: 위 §21개 항목 목록 그대로
- **C·D열 내용 고정** (행7~27):

| 행 | C열 (계획서 기재값) | D열 (검증/기준값) |
|----|--------------------|-------------------|
| 7 | `첨부` 또는 `미첨부` | `관련자료 내 부하계산서(설계서) 수록 여부` |
| 8 | 부하계산서 문서상 건물명 (없으면 `해당없음`) | `설치계획서 건물명: {건물명}` |
| 9 | `명시` / `— (미기재)` / `해당없음`(계산서 무) | `계산방법(프로그램/기준) 명시 필요` |
| 10 | `='건물현황및냉방부하현황'!D16` (기재 RT) | `='건물부하계산서'!B22` (계산서 RT) |
| 11 | `='건물현황및냉방부하현황'!D18` (기재 RTh) | `해당없음` (일부하량은 부하계산서에서 다루지 않음) |
| 12 | 연면적㎡ 숫자 | `='건물현황및냉방부하현황'!D8` |
| 13 | 냉난방면적㎡ 숫자 | `='건물현황및냉방부하현황'!D10` |
| 14 | `{HH}:00 시작` | `22:00 시작 기준` |
| 15 | `='시간대별운전계획(냉방)'!C50` (심야 부하 합계) | `='시간대별운전계획(냉방)'!E50` |
| 16 | `='시간대별운전계획(냉방)'!C51` | `='시간대별운전계획(냉방)'!D51` (주간 최대×60%) |
| 17 | `='시간대별운전계획(냉방)'!C52` | `='시간대별운전계획(냉방)'!D52` (일일×40%) |
| 18 | 심야전력 설비용량 숫자(kW) | `기준 ≥ 20 kW` |
| 19 | 기재 축열률 숫자(%) | `='축냉설비설치계획'!B38` |
| 20 | 상시전력 계약종별 문자열 | `산업용전력(을) 아님 여부` |
| 21 | 기재 입력합계 숫자(kW) | `='축냉설비설치계획'!E15` |
| 22 | `{모드별 용량비 나열: 냉방 {n}% / 제빙 {n}%}` | `기준 ≥ 100% (전 모드)` |
| 23 | `{모드별 용량비 나열: 냉방 {n}% / 제빙 {n}%}` | `기준 < 120% (전 모드)` |
| 24 | 축열조 설계용량 숫자(RTh) | `='시간대별운전계획(냉방)'!C45` (최대 누계) |
| 25 | 축열조 설계용량 숫자(RTh) | `='축열조'!B56` |
| 26 | 난방 총부하 숫자 또는 `해당없음` | `냉방 총부하 (RTh)` 숫자 |
| 27 | `{펌프별 LPM/RT 나열: 이름 값 / 이름 값}` | `기준범위 9~11 LPM/RT` |

- **E열**: 기호 값만 (decision_rules.md 판정표 적용)
- **F열(근거 위치)**: 전 행에 `{파일명} p.{n} ({절 제목})` 기입(v17). 원문이 없으면 `해당없음`.
  설계서·데이터시트는 접두어로 구분(`설계서 {파일명} p.{n}` / `데이터시트 {파일명} p.{n}`)
- **행28~ 추가 발견사항**: decision_rules.md §5 카탈로그에서 발견된 항목 **전부**(체크리스트
  행7~27과 같은 사안이라도 제외하지 않음), 카탈로그 순서대로. A열=`추가`, B열=카탈로그 표준 제목,
  C열=카탈로그 D열 형식 문자열, D열=카탈로그 E열 형식 문자열, E열=`⚠️`, F열=근거 위치
- **`시트확인` 블록**(카탈로그 다음 +1행부터, v17): 다른 시트의 확인사항을 통합한다.
  A열=`시트확인`, B열=항목, C열=계획서 기재, D열=검증값, E열=`⚠️`, F열=근거 위치.
  대상 = 시트3 `[5] 확인사항`, 시트6·7 `[4] 확인사항`, 시트9 산출근거 판정 ⚠️ 등
  (`업체확인필요사항`이 빈 양식이므로 분석 결과는 전부 여기 모인다)
