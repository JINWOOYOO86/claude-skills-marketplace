# Phase 4 — 시간대별 운전계획 검증 (`시간대별운전계획(냉방)` / `시간대별운전계획(난방)`)

> **목적**: 시간대별 냉·난방부하의 적절성을 확인한다. ① 수록 위치 확인 → ② 시간대별 부하량,
> ③ 열원기기(냉동기·히트펌프) 운전계획, ④ 열원기기가 축열조에 넣는 열량, ⑤ 축열조 누적을
> 계산·검증한다.

## 두 가지 대원칙

| 원칙 | 내용 |
|------|------|
| **단위 무변환** | 설치계획서에 적힌 단위를 **그대로 유지**한다. RT면 RT, RTh면 RTh, kW면 kW, kcal/h면 kcal/h. **환산 금지** (RT↔kW 곱셈 금지). 단위는 시트 [1]에 명시한다 |
| **누계 기준시각** | 축열조 누적열량의 기준(0점)은 **축열이 시작되기 직전**이다. 원문 표가 1:00부터 시작하더라도 **축열 시작 시각이 첫 행이 되도록 24시간 표를 회전**시킨 뒤 누적한다 |

누계 점화식: **누계(t) = 누계(t−1) + 열원기기 축열량(t) − 축열조 방열량(t)**, 첫 행 누계 = 축열량 − 방열량.

---

## 작성 방법 — 빌더 스크립트를 쓴다

```bash
# ① 12시트 단일 산출물에 병합 — 기본 (v17)
python scripts/build_ops_plan.py ops_{현장}.json --into "KEPCO_TES_installation_plan_analysis_({파일명}).xlsx"

# ② 운전계획만 단독 검증할 때
python scripts/build_ops_plan.py ops_{현장}.json -o "KEPCO_TES_시간대별운전계획_{현장}.xlsx"
```

- `--into`는 기존 워크북에 `시간대별운전계획(냉방)`·`(난방)` 시트를 갈아끼우고 표준 순서로 정렬한다.
- **난방(또는 냉방) 표가 문서에 없으면** 해당 시트 항목에 `"미수록": true`만 넣는다 →
  `[1]` 블록(수록 위치 `— (미기재)`)만 있는 시트가 생성된다. 표를 지어내지 않으면서
  12시트 구성이 유지된다. `rows`·`units` 등 나머지 키는 넣지 않는다.
- JSON 스키마: `assets/opsplan_schema.example.json` (판독값만 담는다. 작업 폴더에 두고 스킬에 넣지 않는다)
- 냉방·난방 표가 둘 다 있으면 `sheets` 배열에 두 개를 넣는다 → 시트 2개로 생성된다.
- 빌드 시 V1~V14 검증을 파이썬으로도 재수행하고, 불일치가 있으면 **종료코드 1**로 알린다.

## (1) 수록 위치 확인 — 시트 [1] 블록 (행3~7)

| 셀 | 내용 |
|----|------|
| A4 / B4 | `수록 위치` / `{파일명} p.{n} ({절 제목})` |
| A5 / B5 | `표 제목` / 원문 표 제목 그대로 (예: `◇ 최대부하시 운전상황 ◇`) |
| A6 / B6 | `단위` / `부하·운전량 {단위} / 누계 {단위}` |
| A7 / B7 | `축열 시작 시각` / `{HH}:00 (누계 기준시각 = 축열 시작 직전)` |

- 표가 없으면 여기서 끝낸다(B4=`— (미기재)`). 없는 표를 추정해 채우지 않는다.
- 흔한 수록 위치: 본문 「4. 축냉설비 운전계획 — 가. 시간대별 운전계획(축열조 및 냉동기 담당부하)」,
  또는 빙축열/수축열 시스템 설계서의 `최대부하시 운전상황`·`시간대별 부하분담` 표.

## (2)~(5) 시간대별 표 — 시트 [2] 블록 (행10 헤더, 행11~34)

행10 헤더 (단위는 원문 단위를 그대로 넣는다):
`시각 | 총부하 | 열원기기 축열운전 | 열원기기 직접담당 | 축열조 방열 | 축열조 누계(수식) | 부하 정합(수식) | 계획서 기재 누계 | 누계 대조(수식)`

- **행 순서**: 축열 시작 시각이 첫 행. 24시간을 회전시켜 배열한다.
- **A열 시각 라벨은 원문 표기 그대로** 전사한다(`1:00`·`24:00-01:00`·`0-1` 등 문서마다 다름).
  임의로 한 형식으로 통일하지 않는다.
- **수치는 전사만 한다. 추정·보간 금지.** 부하율(%)만 주어지면 `{피크}×{부하율}`로 계산해 넣되
  그 사실을 확인사항에 남긴다.
- **F열 누계**: 첫 행 `=C{r}-E{r}`, 이후 `=F{전행}+C{r}-E{r}` — **수식 그대로**.
- **G열 부하 정합**: `=IF(ABS(B{r}-(D{r}+E{r}))<=0.5,"✅","⚠️")`
  (그 시각 부하는 열원기기 직접담당 + 축열조 방열로 충당돼야 한다)
- **H·I열**: 원문에 누계가 기재돼 있으면 그 값을 H에 전사하고 I에서 재계산 누계와 대조한다.
  원문 누계가 없으면 H·I는 `해당없음`.
- 축열 시간대는 연녹색, 방열 시간대는 연노랑 음영 (빌더가 자동 처리).
- 마지막에 합계행(`=SUM`).

### 열 정의가 양식마다 다르다 — 전사 전에 반드시 판별할 것

| 함정 | 판별법 | 처리 |
|------|--------|------|
| `축열량`·`축냉량` 열이 **시간당 값이 아니라 누계** | 축열 시간대에 값이 계속 증가하고 비고에 `최대축열량` 표기가 붙는다 | 그 열은 `기재누계`로, 시간당 축열은 `히트펌프 축냉(축열)운전` 열에서 가져온다 |
| 축열운전 열이 **총출력(gross)** — 부하 직접담당분을 포함 | 원문 누계가 `축열운전 − 직접담당 − 방열`로 쌓이고, 재계산 누계와의 편차가 **심야 직접담당분 합계와 정확히 일치**한다 | 전사 오류가 아니다. V5·V6·V7·V11 ⚠️의 사유를 확인사항에 적는다 (임의로 값을 빼서 맞추지 않는다) |
| 축열운전에 **음수** 기재 | 누계를 설계용량으로 강제 하향시키려는 표기 | 원문 그대로 전사하고 확인사항에 기재(손실분이면 방열 열로 이관해야 함) |
| 난방표가 **2시간 간격(격행)** 기입 | 홀수 시간대 행이 전 항목 공란 | 공란은 0으로 전사해 24행을 유지하고, 축열량까지 공란인 행은 `기재누계`를 생략(대조 제외) |
| `계` 행이 다른 항목의 합계로 **오기** | 계 행 값이 24행 합계와 다르고, 같은 행 다른 열의 합계와 일치한다 | 전사를 고치지 말고 V1~V4 ⚠️를 확인사항으로 올린다 |

> **축열률의 분모도 양식마다 다르다.** 계획서 기재 축열률은 대개 `축열량 ÷ 주간부하`인데, 이 시트의
> V8은 `방열 합계 ÷ 총부하 합계`로 계산한다. 두 값이 다르면 확인사항에 **산정 기준 차이**로 적는다
> (총부하 기준으로 40% 미만이 되는 사례가 실제로 있다).

## 검증 — 시트 [3] 블록

| 코드 | 검증 |
|------|------|
| V1~V4 | 총부하·축열운전·직접담당·방열 **합계 = 계획서 기재 합계** (전사 누락 검증) |
| V5 | **축열량 합계 = 방열량 합계** (일주기 수지 균형) |
| V6 | **최종 누계 = 0** (기준시각으로 복귀) |
| V7 | **최대 누계 ≤ 축열조 설계용량** (초과 시 ❌) |
| V8 | **축열률 = 방열 합계 ÷ 총부하 합계 ≥ 40%** |
| V9 | 시간대별 최대 총부하 = 계획서 기재 피크부하 |
| V10 | 부하 정합(G열)이 전 시각 ✅ (⚠️ 행수 = 0) |
| V11 | 계획서 기재 누계와 재계산 누계 일치 (⚠️ 행수 = 0) |
| V12 | **심야(22:00~08:00) 부하 합계** — 0이면 심야부하 없음 |
| V13 | **심야 최대부하 ≤ 주간 최대부하 × 60%** (KEPCO 심사항목) |
| V14 | **심야 부하량 ≤ 일일 부하량 × 40%** (KEPCO 심사항목) |

**심야 행 판별**: 빌더가 시각 라벨에서 자동 판별한다 — 구간 표기(`24:00-01:00`, `0-1`, `22:00~23:00`)는
앞 숫자를 시작시로, 단일 표기(`23:00`, `8:00`)는 그 시각을 종료시로 보아 시작시 = t−1로 계산하고,
시작시가 22·23·0~7시면 심야로 잡는다. 판별 결과는 빌드 시 표준출력에 찍히므로 **반드시 눈으로 확인**하고,
틀렸으면 JSON에 `"심야_시각": ["23:00", ...]`으로 직접 지정해 덮어쓴다.

- V1~V4가 어긋나면 **전사 오류를 먼저 의심**하고 원문을 재판독한다.
- V5~V7이 어긋나면 계획서 자체의 오류이므로 [4] 확인사항에 기재한다.
- 심야시간(22:00~08:00) 밖에서 축열하거나, 심야시간 일부를 비워 두면 [4]에 기재한다.

## 다른 시트와의 관계

- 총부하 합계는 시트4 `건물현황및냉방부하현황`의 **최대부하일 부하량 기재값**과 대조한다.
- 체크리스트 행15·16·17(심야부하 판정)은 이 시트의 **V12·V13·V14(행50·51·52)** 를 참조한다.
- 시간대별 최대 총부하는 시트3 `건물부하계산서`의 **냉방 최대부하(RT)**(B22)와 대조한다.
  운전계획이 부하계산서보다 작은 부하로 작성돼 있으면 확인사항에 기재한다.
- 축열조 설계용량은 시트9 `축열조`의 값을 쓴다.
