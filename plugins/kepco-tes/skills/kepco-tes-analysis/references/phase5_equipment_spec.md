## 열원기기 용량 적정성 블록 (행35~50) — 운전모드별 용량 vs 운전량

> **모드별로 따로 본다(v17).** 열원기기는 운전모드마다 능력이 다르므로, 각 능력을 그 모드가
> 실제로 돌아가는 시간대의 **최대 운전량**과 비교한다. 전체 최대냉방부하 하나로 뭉뚱그리면
> 부분축열 설계에서 항상 '용량 부족'이 뜬다.
>
> | 모드 | 비교 대상 운전량 | 시간대 |
> |------|------------------|--------|
> | **제빙** | 열원기기 **축열운전** 최대 | **심야 22:00~08:00** |
> | **냉방** | 열원기기 **직접담당** 최대 | **주간 08:00~22:00** |
> | **난방** | 열원기기 난방운전 최대 | 시트7 난방 운전시간대 |
>
> 판정: **부족 `< 100%` / 정상 `100~120%` / 과다 `≥ 120%`**

```python
ws["A35"] = "[열원기기 용량 적정성 — 운전모드별 용량 vs 운전량]"
ws["A36"] = "열원기기 냉방 총용량(kW)";        ws["B36"] = "=C7"
ws["A37"] = "열원기기 냉방 총용량(RT)";        ws["B37"] = "=B36/3.517"
ws["A38"] = "열원기기 제빙 총용량(kW)";        ws["B38"] = "=C13"
ws["A39"] = "열원기기 제빙 총용량(RT)";        ws["B39"] = "=B38/3.517"
ws["A40"] = "열원기기 난방 총용량(kW)";        ws["B40"] = "=C10"      # 없으면 해당없음
ws["A41"] = "열원기기 난방 총용량(RT)";        ws["B41"] = "=B40/3.517"  # 없으면 해당없음
ws["A42"] = "주간(08:00~22:00) 최대 운전량(RT)"; ws["B42"] = 주간운전RT   # 값
ws["A43"] = "심야(22:00~08:00) 최대 운전량(RT)"; ws["B43"] = 심야운전RT   # 값
ws["A44"] = "난방 최대 운전량(RT)";            ws["B44"] = 난방운전RT     # 값 / 해당없음
ws["A45"] = "냉방 용량비 (냉방용량/주간운전)";   ws["B45"] = "=B37/B42"
ws["A46"] = "냉방 판정 (부족<100% / 과다≥120%)"
ws["B46"] = '=IF(B45<1,"❌ 용량 부족",IF(B45>=1.2,"❌ 과다 산정","✅ 정상"))'
ws["A47"] = "제빙 용량비 (제빙용량/심야운전)";   ws["B47"] = "=B39/B43"
ws["A48"] = "제빙 판정 (부족<100% / 과다≥120%)"
ws["B48"] = '=IF(B47<1,"❌ 용량 부족",IF(B47>=1.2,"❌ 과다 산정","✅ 정상"))'
ws["A49"] = "난방 용량비 (난방용량/난방운전)";   ws["B49"] = "=B41/B44"     # 없으면 해당없음
ws["A50"] = "난방 판정 (부족<100% / 과다≥120%)"
ws["B50"] = '=IF(B49<1,"❌ 용량 부족",IF(B49>=1.2,"❌ 과다 산정","✅ 정상"))'  # 없으면 해당없음
```

**B42·B43·B44(최대 운전량) 값 규칙 — 고정**
- 시트6·7 `시간대별운전계획`에서 읽는다. 시트6·7은 원문 단위 그대로이므로(Phase 4 무변환)
  **RT로 환산해 넣는다**(`USRt`=`RT` 그대로 / `kW`÷3.517 / `kcal/h`÷3,024). 소수 1자리.
  - **B42 주간 운전량** = 08:00~22:00 행의 `열원기기 직접담당` 최대
  - **B43 심야 운전량** = 22:00~08:00 행의 `열원기기 축열운전` 최대
  - **B44 난방 운전량** = 시트7의 열원기기 운전 최대 (냉열전용이면 `해당없음`)
- 해당 모드가 없는 기기(제빙 없음·난방 없음)는 용량·운전량·용량비·판정을 전부 `해당없음`.
- 시간대별 표가 없으면(K5) 운전량을 알 수 없으므로 해당 모드 행을 `확인불가`로 두고
  업체확인 카탈로그 K5로 기재한다. **추정값을 넣지 않는다.**

## 축열능력 계산 검증 (행25~, 반드시 포함)

> **핵심 원칙**: 내부체적을 **순환매체량(브라인/물 등)** 과 **축열매체량(얼음/물/PCM 등)** 으로
> **구분**하여 각각 계산한다. 각 매체의 **온도범위·유체종류·축열매체 종류**를 먼저 확인하고,
> 해당 **물성치(밀도 ρ, 비열 Cp, 잠열 L)** 를 조회하여 **현열축열량·잠열축열량을 각각** 계산한 뒤
> 합산값을 설치계획서 값과 **비교검증**한다. 사용한 **수식까지 그대로** 엑셀에 표시한다.

### 물성치 — `decision_rules.md` §3.1 고정값 사용 (조회 금지, 임의 변경 금지)
> 축열매체 체적·순환매체 체적·잠열/현열 산식도 decision_rules §3의 표준 계산법을 그대로 따른다.

### ▶ 수축열식 (현열만)
```python

ws["A27"] = "온도범위(입구→출구, ℃)";   ws["B27"] = f"{입구T}→{출구T}"
ws["A28"] = "내부체적 V (㎥)";          ws["B28"] = 내부체적
ws["A29"] = "물 밀도 ρ (kg/㎥)";        ws["B29"] = 1000
ws["A30"] = "물 비열 Cp (kcal/kg℃)";    ws["B30"] = 1.0
ws["A31"] = "온도차 ΔT (℃)";            ws["B31"] = f"=ABS({입구T}-{출구T})"
ws["A32"] = "현열축열량 (kcal)";        ws["B32"] = "=B28*B29*B30*B31"
ws["A33"] = "현열축열량 (RTh)";         ws["B33"] = "=B32/3024"

# === 합계 및 비교검증 — 축열형식과 무관하게 행55~60 고정 (체크리스트가 B56을 참조) ===
ws["A55"] = "[합계 및 비교검증]"
ws["A56"] = "총 축열능력 계산값 (RTh)";        ws["B56"] = "=B33"      # 수축열식은 현열만
ws["A57"] = "설계(설치계획서) 축열능력 (RTh)"; ws["B57"] = 설계RTh
ws["A58"] = "차이 (RTh)";               ws["B58"] = "=B56-B57"
ws["A59"] = "차이율 (%)";               ws["B59"] = "=ABS(B58/B57)*100"
ws["A60"] = "검증 판정";                ws["B60"] = '=IF(B59<10,"✅","⚠️")'
```
> **행55~60(합계·비교검증)은 축열형식이 무엇이든 같은 위치**에 둔다. 체크리스트 행25가
> `='축열조'!B56`을 참조하므로, 수축열식이라고 해서 B33에서 끝내면 교차참조가 깨진다.

## 축열조 용량 계산자료 유무 (행62~72, 반드시 포함)

> 위 검증 블록은 **스킬이 스스로 재계산**하는 절차다. 그것만으로는 *계획서가 산출근거를
> 제출했는지*를 알 수 없다(결과값만 적어 내도 재계산이 맞으면 ✅가 나온다).
> 축열능력은 **크기 → 축열매체·순환매체 종류와 체적 → 운영온도 → 제빙율 → 잠열·현열 계산**
> 순으로 도출되므로, **그 입력값이 항목별로 수록됐는지 하나씩 확인**한다.

```python
ws["A62"] = "[축열조 용량 계산자료 — 산출 입력값별 수록 여부]"
ws["A63"] = "① 축열조 내부치수"          # B=수록/미수록(+값), F=출처
ws["A64"] = "② 축열매체 종류·체적"
ws["A65"] = "③ 순환매체 종류·체적"
ws["A66"] = "④ 운영온도(주간/야간)"
ws["A67"] = "⑤ 제빙율 IPF"
ws["A68"] = "⑥ 잠열·현열 계산식"          # ← 이것이 없으면 '계산자료'라 부를 수 없다
ws["A69"] = "계산자료 유무"               # 유 / 무
ws["A70"] = "수록 형태"                  # 아래 4개 문자열 중 하나
ws["A71"] = "산출근거 판정"               # ✅ / ⚠️
ws["A72"] = "→ 축열능력 검증은 위 [합계 및 비교검증] 블록의 재계산으로 대체"
```

**B70 `{수록 형태}` — 이 4개만 사용 (자유 서술 금지, R2)**

| ①~⑤ 입력값 | ⑥ 계산식 | `{수록 형태}` | B69 | B71 |
|------------|----------|---------------|-----|-----|
| 수록 | 수록 | `용량 산출내역 수록` | `유` | `✅` |
| 수록 | 미수록, 대수 산정만 | `대수 산정만 수록` | `유` | `⚠️` |
| 일부/미수록 | 미수록 | `결과값만 수록` | `무` | `⚠️` |
| 미수록 | 미수록 | `미수록` | `무` | `⚠️` |

- **`대수 산정만 수록`이 빙축열에서 가장 흔하다.** 대당축열량(예: #950 = 813 USRt·h)은
  제조사 모델 인증값이라 계획서에 잠열·현열 산출식이 없는 것이 보통이다. 그래도 `⚠️`로 두고
  카탈로그 T3로 올려 산출근거를 요청한다(검증 자체는 위 블록의 재계산으로 이미 수행됨).
- 행73 이후 사용 금지.

> 잠열만으로 설계값을 설명하는 계획서가 많다. 잠열단독(B37)이 설계값과 맞고 합계(B56)가 크게 벗어나면
> **시트에 참고 행을 추가하지 말고**(R1) `업체확인필요사항`에 산정 기준 차이로 기재한다.

# (3) 펌프 시트 (`펌프`)

> **블록 행 배치·항목·순서는 아래 시트10 셀 맵을 그대로 사용한다.**
> 대상은 순환펌프(냉수/브라인/냉각수)만 — 링블로워·보충수펌프는 이 시트에 넣지 않는다.
> **담당 냉동능력(RT) 분모는 `decision_rules.md` §2 정의를 그대로 사용**하고,
> 다른 분모의 참고 계산 행을 추가하지 않는다.

## 순환유량 적정성 검증 — LPM/RT (반드시 포함)
> RT당 순환유량을 계산하고 기준 **10 LPM/RT**에서 **±10%(9~11)** 벗어나면
> `체크리스트`·`업체확인필요사항`에 포함한다. 수식 그대로 표시.

## 펌프별 블록 — 아래 시트10 셀 맵을 그대로 사용

블록 시작행 `s = 3 + 14×(i−1)`, 13행 + 빈행 1행. 항목·오프셋·수식은 시트10 맵이 규정하므로
**여기에 별도 항목표를 두지 않는다**(이중 규격 방지). 핵심 검증만 다시 확인한다:

- `s+7` LPM/RT = `=B{s+5}/B{s+6}` — **수식 그대로**
- `s+9` 판정 = `=IF(AND(B{s+7}>=9,B{s+7}<=11),"✅","⚠️ 기준이탈")`
- 이탈 펌프가 하나라도 있으면 체크리스트 행27 = ⚠️, 업체확인 카탈로그 P2 기재
- 블록 외 참고 행·요약 행 추가 금지

# (6) 감소전력계산 시트 (`감소전력계산`) — v17

> 근거: **수요관리업무처리지침(H0-영업-지침-0032) 제2장 1.나 설치지원금의 산정** p.81~83.
> 계획서 기재 감소전력을 그대로 받지 않고 **지침 산식으로 재산정해 대조**한다.

```
감소전력(kW) = 냉방에 이용가능한 축열조 저장열량(kcal) ÷ (표준냉방시간 10h × 3,024 kcal/kWh)
             (1kW 미만은 소수점 첫째자리에서 반올림)
```

**'냉방에 이용가능한 축열조 저장열량'은 아래 셋 중 가장 적은 값** (지침 p.81 (3)):

| 코드 | 산정 방법 |
|------|-----------|
| (가) | 축방열방법별 **축열조 저장가능열량 × 95%** |
| (나) | **선정된 열원기기 축열열량 × 90%** (= 제빙능력 RT × 3,024 × 심야 축열시간) |
| (다) | **고객이 제시한 축열가능열량** |

### 축열조 저장가능열량 = 잠열량 + 현열량 (지침 p.82 ①)

- 잠열량 = 상변화 축열매체 질량(kg) × 단위질량당 잠열(kcal/kg)
- 현열량 = {(축열매체 질량 × 비열) + (순환매체 질량 × 비열)} × **이용온도차**

**축방열방법별 질량 산출** (지침 p.82~83 ②) — 빙축열(관외착빙형) 기준:
- 축열매체 질량 = [바닥면적(㎡) × 축열매체높이(m) − 코일·헤더 체적(㎥)] × 1,000 × 물 비중
- 상변화 축열매체 질량 = 축열매체 질량 × **IPF(%)/100** (IPF는 설치계획서 기재값)
- 순환매체 질량 = 코일·헤더 총 내부체적(㎥) × 순환매체 비중 × 1,000

**이용온도차 (지침 p.83 ③, 고정값)**

| 축냉방식 | 물 또는 잠열재 | 브라인 |
|----------|---------------|--------|
| 빙축열식 | 5℃ | **9.5℃** (아이스슬러리형은 8℃) |
| 수축열식 | 10℃ | — |
| 상온잠열축열식 | 12℃ − 상변화온도 | — |

## 설비 목록 누락 검증 (시트5 행53~, v17)

> `축냉설비설치계획` 1) 설비 목록에 있는 기기는 **빠짐없이 대응 규격 시트를 가져야** 한다.
> 목록만 있고 시트가 없으면 그 기기는 검증되지 않은 채 지나간다.

**대조 방법 — 수식으로 실제 값을 끌어온다(존재 여부만 보지 않는다)**
- D열이 대응 시트의 제작사·모델 셀을 참조한다. 시트가 없으면 `#REF!`, 비어 있으면 공란 →
  E열 판정이 `❌ 시트 누락`이 된다.
- B열은 설비 목록의 비고(제작사·모델)를 `=G{행}`으로 끌어와 **같은 기기를 가리키는지** 눈으로 대조한다.

**양방향으로 본다**
| 방향 | 확인 내용 |
|------|-----------|
| 목록 → 시트 | 설비 목록의 각 행에 대응 시트가 있는가 (E열 판정) |
| 시트 → 목록 | 시트는 있는데 설비 목록에 없는 기기가 있는가 (마지막 행 `역방향 확인`) |

- **축열조는 설비 목록(행4~13)이 아니라 `2) 냉온열축열조` 블록에 기재되는 것이 정상**이다.
  역방향 확인 행에 그 사실을 적어 '누락'으로 오판하지 않게 한다.
- 누락이 하나라도 있으면 카탈로그 **E4**로 기재한다.
