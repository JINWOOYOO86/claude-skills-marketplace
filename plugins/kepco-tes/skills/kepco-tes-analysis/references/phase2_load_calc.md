# Phase 2 — 건물부하계산서 시트 (`건물부하계산서`)

> **목적**: 건물 각 공간의 **면적**과 **냉·난방부하**를 확인하고, 이를 **합산해 최대부하율**을 도출한 뒤
> 설치계획서 기재값과 대조한다. 그 외 정보(건물명·계산방법·작성주체 등)는 **기재하지 않는다**.
> 시트3·시트4는 항상 별도 시트. 합치지 않는다.

## 진행 순서 — 이 순서 그대로, 앞 단계가 실패하면 뒤 단계 없음

| 단계 | 내용 | 실패 시 |
|------|------|---------|
| **(1)** | 부하계산서 **유무** 확인 | **`무`이면 (2)·(3)을 수행하지 않는다** — 행8 이하 공란, 확인사항만 기재 |
| **(2)** | 각 공간의 **면적·냉방부하·난방부하** 전사 → **면적 계산검증** | 면적이 없으면 부하만 전사하고 부하율은 `산출불가` |
| **(3)** | **합산 → 최대부하율** 도출, 설치계획서 기재값과 대조 | — |

- **'유'의 기준**: 부하를 **산출한 내역**(실별·존별·층별 집계표, 부하계산 프로그램 출력물,
  `면적 × 단위부하` 산출표, 건물 PEAK LOAD 표 등)이 실제로 수록된 경우.
- **'무'로 판정**: 목차·간지만 있고 실물이 없는 경우, 결과값만 옮겨 적힌 경우,
  빙축열 시스템 설계서·장비선정서로 대체된 경우(부하가 이미 주어진 입력값). → B4=`무`, [4]에 기재.

## 작성 방법 — 빌더 스크립트를 쓴다 (직접 openpyxl 코딩 금지)

셀맵·수식·검증 로직은 `scripts/build_load_calc.py`가 고정한다. **문서에서 판독한 값만 JSON에 담고**
스크립트로 빌드한다 — 매번 손으로 코딩하면 셀맵이 조금씩 어긋나 재현성이 깨진다(R1).

```bash
# 판독값 JSON은 산출물과 같은 작업 폴더에 둔다 (스킬 저장소에 넣지 않는다)

# ① 12시트 단일 산출물에 병합 — 기본 (v17)
python scripts/build_load_calc.py loadcalc_{현장}.json --into "KEPCO_TES_installation_plan_analysis_({파일명}).xlsx"

# ② 건물부하계산서만 단독 검증할 때
python scripts/build_load_calc.py loadcalc_{현장}.json -o "KEPCO_TES_건물부하계산서_{현장}.xlsx"
```

- `--into`는 기존 워크북을 열어 `건물부하계산서`(+`실별부하집계`) 시트를 **갈아끼우고**
  표준 시트 순서로 정렬한 뒤 저장한다. 여러 번 돌려도 시트가 중복되지 않는다.
- **`--into` 모드에서는 아래 `[5] 확인사항` 블록을 생략**한다(시트12 `업체확인필요사항`이 전담).

- JSON 스키마·필드 설명: `assets/loadcalc_schema.example.json`
- 빌드 시 **실별 상세표의 열별 합계를 계산서 기재 Grand Total과 자동 대조**하고, 한 열이라도
  불일치하면 종료코드 1로 알린다 → **전 열 ✅가 될 때까지 재판독**한다.
- 아래 셀 맵은 이 스크립트가 생성하는 결과의 규격이며, 스크립트 수정 시 함께 갱신한다.

## 시트 고정 셀 맵

### [1] 부하계산서 유무 (행3~7)

| 셀 | 내용 |
|----|------|
| A1 | `건물부하계산서 검토` |
| A3 | `[1] 부하계산서 유무` |
| A4 / B4 | `부하계산서 유무` / `유` 또는 `무` |
| A5 / B5 | `수록 위치` / `{문서명} p.{n}~{n}` (무이면 `해당없음`) |
| A6 / B6 | `수록 형태` / 표 구성 한 줄 요약 (무이면 `해당없음`) |
| A7 / B7 | `부하 단위` / `W` 또는 `kcal/h` (무이면 `해당없음`) |

> **B4=`무`이면 여기서 끝낸다.** 행8 이하는 비우고 [5] 확인사항만 행10부터 작성한다.

### [2] 면적 계산검증 (행9~16, 행번호 고정)

A9=`[2] 면적 계산검증`, 행10 헤더: A=`항목` B=`계산값(수식)` C=`단위` D=`대조값` E=`판정`

| 행 | A열 | B열(수식) | C열 | D열(대조값) | E열(판정) |
|----|-----|-----------|-----|-------------|-----------|
| 11 | 공간별 냉방면적 합계 | `=SUM(C29:C300)` | `㎡` | 계산서 기재 합계(숫자) | `=IF(ABS(B11-D11)<=1,"✅","⚠️")` |
| 12 | 공간별 난방면적 합계 | `=SUM(F29:F300)` | `㎡` | 계산서 기재 합계(숫자) | `=IF(ABS(B12-D12)<=1,"✅","⚠️")` |
| 13 | 냉방면적 − 난방면적 | `=B11-B12` | `㎡` | `0` | `=IF(ABS(B13-D13)<=1,"✅","⚠️")` |
| 14 | 냉난방면적 (설치계획서 대조) | `=B11` | `㎡` | 설치계획서 기재 ㎡ | `=IF(ABS(B14-D14)<=1,"✅","❌")` |
| 15 | 냉난방면적 (평 환산) | `=B11/3.30579` | `평` | 설치계획서 기재 평 | `=IF(ABS(B15-D15)<=1,"✅","⚠️")` |
| 16 | 연면적 대비 냉난방면적 비율 | `=B11/D16` | `%` | 연면적 ㎡(숫자) | `해당없음` |

- **검증의 뜻**: 행11·12는 **전사 누락**(공간별 면적을 다 옮겼는지), 행13은 **냉·난방 대상 면적 차이**,
  행14·15는 **설치계획서 기재값과의 일치**, 행16은 **과다·과소 여부 참고지표**를 본다.
- 면적이 하나도 없는 계산서면 행11~16 B·D·E열 전부 `해당없음`, [5]에 "면적 미기재"로 기재.
- 일부만 있으면 있는 행만 합계되므로 행11 E열은 `⚠️`(값)로 두고 [5]에 기재한다.
- 행16 D열이 없으면(연면적 미기재) B16·E16 `해당없음`.

### [3] 최대부하 및 최대부하율 (행18~25, 행번호 고정)

A18=`[3] 최대부하 및 최대부하율`, 행19 헤더: A=`항목` B=`계산값(수식)` C=`단위` D=`설치계획서 기재값` E=`판정`

| 행 | A열 | B열(수식) | C열 | D열 | E열 |
|----|-----|-----------|-----|-----|-----|
| 20 | 냉방 최대부하 합계 | `=SUM(D29:D300)` | B7 단위 | 계산서 기재 합계 | `=IF(ABS(B20-D20)<=1,"✅","⚠️")` |
| 21 | 냉방 최대부하율 | `=B20/B11` | `{B7단위}/㎡` | `해당없음` | `해당없음` |
| 22 | 냉방 최대부하(RT) | `=B20/3517` 또는 `=B20/3024` | `RT` | 기재 RT(숫자) | `=IF(ABS(B22-D22)<=D22*0.05,"✅","❌")` |
| 23 | 난방 최대부하 합계 | `=SUM(G29:G300)` | B7 단위 | 계산서 기재 합계 | `=IF(ABS(B23-D23)<=1,"✅","⚠️")` |
| 24 | 난방 최대부하율 | `=B23/B12` | `{B7단위}/㎡` | `해당없음` | `해당없음` |
| 25 | 난방 최대부하(RT) | `=B23/3517` 또는 `=B23/3024` | `RT` | 기재 RT(숫자) | `=IF(ABS(B25-D25)<=MAX(1,D25*0.05),"✅","❌")` |

- **RT 환산 분모 고정**: B7=`W`면 **3517**, B7=`kcal/h`면 **3024** (1 RT = 3,517 W = 3,024 kcal/h).
- 부하율(행21·24)은 **[2]의 면적 합계 B11·B12를 분모로** 쓴다(면적 검증을 통과한 값이어야 한다).
- 난방부하가 계산서에 없으면(냉열전용) 행23~25의 B·D·E열 전부 `해당없음`.
- 면적이 미기재면 행21·24 B열은 `산출불가`(문자).

### [4] 공간별 면적 및 냉난방부하 (A27 소제목, 행28 헤더, 행29~ 데이터)

행28 헤더: `A=공간(실/존/계통)` `B=실수` `C=냉방면적(㎡)` `D=냉방부하` `E=냉방부하율(W/㎡)`
`F=난방면적(㎡)` `G=난방부하` `H=난방부하율(W/㎡)` `I=단위부하(기재)` `J=부하 재계산(수식)` `K=재계산 판정`

- 행29부터 **부하계산서의 집계 단위 그대로** 한 행씩 전사한다(실별 표면 실별, 존별이면 존별, 계통별이면 계통별).
- E열 = `=D{행}/C{행}`, H열 = `=G{행}/F{행}` (수식 그대로).
- **면적 × 단위부하 방식 계산서**(`{면적}㎡ × {단위부하}kcal/㎡h`)면 I·J·K를 채운다:
  I=계산서 기재 단위부하, J=`=C{행}*I{행}`, K=`=IF(ABS(J{행}-D{행})<=MAX(1,D{행}*0.01),"✅","⚠️")`
  → **면적과 부하의 정합을 행 단위로 검증**한다. 요소별(프로그램) 계산서면 I·J·K는 `해당없음`.
- **계산서의 소계·합계·Grand Total 행은 데이터 행으로 넣지 않는다**(이중 계상 방지 — 대조는 [2]·[3]의 D열).
- 면적이 없는 행은 C·F열 `— (미기재)` (SUM은 문자를 무시하므로 면적이 있는 행만 합계된다).

## 상세 실별 부하표가 있으면 — 별도 시트 `실별부하집계` 에 **전량 전사**

계산서에 실별(층·실 단위) 부하 집계표가 있으면 **요약만 옮기지 말고 원문 표를 통째로 전사**한다.
[4]의 공간별 표(집계 단위)와 별개로, 이 시트가 **판독 정확도를 스스로 증명하는 장치**가 된다.

- **행1** 제목 = `{절 번호} 실별 냉난방 부하 집계 (설치계획서 p.{n}~{n}, {문서명})`
- **행3** 헤더 = 원문 열 구성 그대로 + `합계반영` + `비고`
  (예: `No. | 실명 | Q'ty | 8시 | 10시 | 12시 | 14시 | 16시 | 18시 | 20시 | 난방(W) | 합계반영 | 비고`)
- **행4~** 원문 행을 **한 행도 빼지 않고** 전사한다. 원문의 `-`(미산출)는 문자 `-` 그대로.
  **중복 기재된 행·소계 행도 지우지 말고 그대로 전사**하고, `합계반영` 열에 `0`을 넣어 합계에서만 제외한다
  (그 외 행은 `1`). 제외한 행은 `비고`에 사유를 적고 셀을 음영 처리한다.
- **검증 블록** (데이터 마지막 행 + 2행부터):

| 행 | 내용 |
|----|------|
| 소제목 | `[검증] 실별 합계 vs 계산서 기재 Grand Total` |
| 헤더 | `구분` + 원문 부하 열 이름 |
| 실별 합계 | 열마다 `=SUMIF($L$4:$L${마지막},1,{열}4:{열}{마지막})/1000` (kW 표기 시 `/1000`) |
| 계산서 기재 Grand Total | 원문 합계행 값(숫자) |
| 판정 | `=IF(ABS({합계}-{기재})<=0.2,"✅","⚠️")` (kW 기준 ±0.2) |
| 실수 대조 | `=SUMIF($L$4:$L${마지막},1,$C$4:$C${마지막})` vs 원문 실수, `=IF(...,"✅","⚠️")` |

- **`SUMIF`를 쓴다** — `SUMPRODUCT`는 합계범위의 `-`(텍스트)에서 `#VALUE!`가 난다.
- **전 열이 ✅가 될 때까지 재판독한다.** 한 열이라도 ⚠️면 그 열을 다시 읽어 오탈자를 찾는다
  (원문 자체의 중복·누락이 원인이면 `합계반영`=0으로 처리하고 [5] 확인사항에 기재).
- 원문에 실별 **면적** 열이 없으면 그 사실을 [5] 확인사항에 기재한다(실 단위 면적·부하 정합 검증 불가).

### [5] 확인사항 (마지막 데이터 행 + 2행부터, B4=`무`이면 행10부터)

헤더: `A=번호` `B=항목` `C=부하계산서` `D=설치계획서` `E=확인요청 내용`
- [2]·[3]·[4]에서 ⚠️/❌가 나온 항목, 부하계산서 `무`, 면적 미기재를 한 행씩 기재한다.
- **12시트 전체 산출물에서는 이 블록을 생략**하고 시트12 `업체확인필요사항`에만 기재한다.
  건물부하계산서만 단독으로 검증하는 파일에서는 이 블록을 쓴다.
