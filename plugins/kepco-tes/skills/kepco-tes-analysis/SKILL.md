# KEPCO 심야전력 축냉설비 설치계획서 분석 스킬 (v17)

> 현재 규격: **건물부하계산서·시간대별운전계획은 판독값 JSON + 빌더**로 만들어
> `--into`로 **분석 엑셀 한 개**에 병합한다(R8).
> 개정 이력은 `CHANGELOG.md` 참조.

## 파일 구성

| 파일 | 역할 | 로드 시점 |
|------|------|-----------|
| `SKILL.md` | 오케스트레이션·재현성 규칙(R1~R8)·전역 표기/결측 규칙·시트12 | 항상 |
| `references/decision_rules.md` | 판정 조건표·교차검증 카탈로그·물성치 고정값 | Phase 0 |
| `references/phase1_checklist.md` | 21개 심사항목 + **시트1·2 셀 맵** | Phase 1 |
| `references/phase2_load_calc.md` | 건물부하계산서 절차 + **시트3 셀 맵** | Phase 2 |
| `references/phase3_building_status.md` | **시트4 셀 맵** | Phase 3 |
| `references/phase4_operation_plan.md` | 시간대별 운전계획 절차 + **시트6·7 셀 맵** | Phase 4 |
| `references/phase5_equipment_spec.md` | 설비 규격 계산 + **시트5·8·9·10·11 셀 맵** | Phase 5 |
| `scripts/build_load_calc.py` | 시트3 빌더 (판독값 JSON 입력) | Phase 2 |
| `scripts/build_ops_plan.py` | 시트6·7 빌더 (판독값 JSON 입력) | Phase 4 |
| `scripts/build_survey.py` | 여러 현장 정리표 빌더 (JSON 입력) | 일괄 검토 시 |
| `scripts/scan_pdf.py` | PDF 텍스트 레이어·키워드 1차 스캔 | 자료 위치 탐색 |
| `scripts/verify_output.py` | **산출물 규격 검사 16종** (아래 필수 검증 자동화) | 빌드 마지막 단계 |
| `scripts/verify_repro.py` | 산출물 재현성 검증 (셀·sha256) | 산출 후 점검 |
| `scripts/_xlsx_det.py` | 바이트 동일 저장(타임스탬프 정규화) | 빌더가 사용 |
| `assets/*.schema.example.json` | 빌더 입력 JSON 스키마 예시 | 빌더 작성 시 |

> **시트 하나 = 문서 하나.** 셀 맵은 위 표의 담당 문서만 규정한다(이중 규정 금지).
> **도구는 스킬, 판독값은 작업 폴더.** 문서에서 읽어낸 값은 작업 폴더의 JSON
> (`loadcalc_*.json` / `ops_*.json` / `survey_*.json`)에 두고 스킬에 넣지 않는다(R8).

### 여러 현장을 일괄 검토할 때 (사외전문가 활동 등)

```bash
python scripts/scan_pdf.py "{모음폴더}" --keys "시간대별,운전계획"   # ① 자료 위치 후보 추리기
#   → 적중 페이지를 실제로 열어 확인한 뒤 현장별 판독값 JSON 작성
python scripts/build_load_calc.py loadcalc_{현장}.json --into "$OUT"  # ② 현장별 산출(단일 파일)
python scripts/build_ops_plan.py  ops_{현장}.json      --into "$OUT"
python scripts/build_survey.py    survey_{주제}.json                 # ③ 현장 비교 정리표
python scripts/verify_repro.py    "{모음폴더}"                       # ④ 재현성 점검(전건 '동일')
```

## 사전 준비

- 설치계획서 파일 첨부 필수 (없으면 요청)
- xlsx 스킬 + openpyxl로 엑셀 생성
- **건물부하계산서 시트는 `scripts/build_load_calc.py`, 시간대별 운전계획은 `scripts/build_ops_plan.py`로
  빌드**한다(판독값 JSON 입력). 직접 openpyxl 코딩 금지 — 셀맵 이탈·재현성 붕괴의 원인
- **엑셀파일명**: `KEPCO_TES_installation_plan_analysis_(파일명).xlsx`
- **엑셀 양식**: 각 Phase 참조 문서의 **고정 셀 맵**을 그대로 따른다
  (시트 구성·항목명·행번호·열 순서·제목 문구 임의 변경 금지)

## 산출물은 엑셀 **한 개** — 3단계 병합 빌드 (v17)

작업 폴더에 남는 엑셀은 **12시트 분석 파일 하나뿐**이다. 빌더가 별도 파일을 만들지 않고
`--into`로 같은 워크북에 자기 시트를 끼워 넣는다.

```bash
OUT="KEPCO_TES_installation_plan_analysis_({파일명}).xlsx"

python build_{현장}.py "$OUT"                                  # ① 시트1·2·4·5·8~12 (9개)
python scripts/build_load_calc.py loadcalc_{현장}.json --into "$OUT"   # ② 시트3 (+실별부하집계)
python scripts/build_ops_plan.py  ops_{현장}.json      --into "$OUT"   # ③ 시트6·7
python scripts/verify_output.py   "$OUT"                              # ④ 규격 검사 (실패 시 종료코드 1)
```

- ①은 현장별로 작성하는 빌드 스크립트다. **시트3·6·7은 만들지 않는다**(②③이 넣는다).
  스킬 저장소가 아니라 **임시 폴더**에 두어 작업 폴더를 어지럽히지 않는다.
- ②③은 `replace_sheet()`로 동명 시트를 지우고 다시 만들어 **몇 번을 돌려도 중복이 없다**(멱등).
- 삽입 후 `order_sheets()`가 `_xlsx_det.SHEET_ORDER` 기준으로 정렬하므로
  **실행 순서와 무관하게** 항상 같은 12시트 배열이 된다.
- `--into` 없이 실행하면 종전처럼 단독 파일이 나온다(건물부하계산서만 따로 검증할 때).
- **`--into` 모드에서는 시트3의 `[5] 확인사항` 블록을 생략**한다 — 확인사항은 시트12가 전담한다.
- 난방 운전계획표가 문서에 없으면 ops JSON의 난방 시트에 `"미수록": true`를 넣는다.
  `[1]` 블록만 있는 시트가 생성되어 표를 지어내지 않으면서 12시트 구성이 유지된다.

> **작업 폴더에 남는 것**: 분석 엑셀 1개 + 판독값 JSON(`loadcalc_*.json`·`ops_*.json`).
> JSON은 R8 재현성의 원천이므로 **지우지 않는다**(엑셀은 이 JSON에서 언제든 재생성된다).

## 재현성 규칙 (R1~R8) — 항상 적용, 위반 금지

같은 입력에 대해 몇 번을 실행해도 **동일한 엑셀**이 나와야 한다.

| 규칙 | 내용 |
|------|------|
| R1 템플릿 고정 | 각 시트는 **담당 Phase 문서의 셀 맵** 그대로(시트1·2=phase1 / 시트3=phase2 / 시트4=phase3 / 시트6·7=phase4 / 시트5·8~11=phase5 / 시트12=이 문서). 지정되지 않은 셀 사용 금지 |
| R2 자유서술 금지 | 참고·※주석·부연 설명 셀 추가 금지. 표현은 템플릿 형식 문자열만 사용 |
| R3 판단 고정 | 판정·채택·계산법은 `references/decision_rules.md`를 따른다. 임의 판단 금지 |
| R4 확인사항 고정 | 체크리스트 행28~ 추가 발견사항은 decision_rules의 카탈로그 항목만, 카탈로그 순서로. `업체확인필요사항`은 **양식만**(v17) |
| R5 수식/값 구분 | 템플릿이 수식으로 지정한 셀은 수식, 값으로 지정한 셀은 값. 혼용 금지 |
| R6 반올림 고정 | 환산 표시값 소수 1자리, %·LPM/RT 소수 1자리. 수식 셀은 반올림하지 않음 |
| R7 무인모드 | 사용자에게 질문하지 않는다. 애매한 값은 decision_rules §6 채택 규칙 적용 후 체크리스트 행28~에 기록 |
| R8 판독값 분리 | 시트3·시트6·7은 **판독값 JSON + 범용 빌더**로 만들어 `--into`로 분석 워크북 하나에 병합한다(v17 — 산출 엑셀은 항상 1개). 판독값이 JSON에 고정되므로 재실행 시 비전 재판독이 불필요하고, 빌더가 `_xlsx_det.save_deterministic()`으로 저장해 **바이트 동일**이 보장된다. 재판독이 필요하면 빌더의 합계 대조 검증이 편차를 잡아낸다 |

## 판독값 분리 (R8) — 스캔 문서 재현성의 핵심

스캔 이미지 PDF는 비전 재판독 시 자릿수 편차가 생겨 실행마다 값이 달라진다. 그래서
**판독값은 작업 폴더의 JSON에 고정**하고, **셀맵·수식·검증은 스킬의 범용 빌더가 담당**한다.

```
작업 폴더   loadcalc_{현장}.json / ops_{현장}.json    ← 문서에서 읽어낸 값만
스킬        scripts/build_load_calc.py               ← 시트3 셀맵·수식·검증
            scripts/build_ops_plan.py                ← 시트6·7 셀맵·수식·검증
            scripts/_xlsx_det.py                     ← 타임스탬프 정규화(바이트 동일)
```

- **재실행 시 문서를 다시 읽지 않는다.** JSON만 있으면 언제든 같은 엑셀이 나온다.
- **템플릿을 개정해도 낡지 않는다.** 빌더를 고치면 기존 JSON 전부가 새 규격으로 재생성된다.
- **판독 오류는 빌더가 잡는다.** 실별 합계·계 행 대조가 어긋나면 종료코드 1로 알리므로,
  재판독이 필요한 상황을 사람이 놓치지 않는다.
- 검증 실적: 22개 산출물을 3회 독립 실행 → 셀 22/22, sha256 22/22 동일.

### 신규 문서 절차
Phase 0~5로 분석하면서 판독값을 JSON에 적고 빌더를 실행한다. 빌더의 자체 검증이 전부
통과하면 그 JSON이 곧 그 문서의 고정 대장이 된다(별도 등록 절차 없음).

## `업체확인필요사항` 시트 (항상 추가, **맨 앞 시트**) — 양식만 생성

업체에 보낼 확인 요청을 **사용자가 직접 골라 작성**하는 시트다. 스킬은 **머리글 양식만** 만들고
**데이터 행은 채우지 않는다**(v17). 분석이 찾아낸 발견사항은 `체크리스트` 시트가 전담한다
— 행7~27의 ⚠️/❌ 판정과 행28~ 추가 발견사항(교차검증 카탈로그)을 보고 사용자가 옮겨 적는다.

```python
ws_q = wb.create_sheet("업체확인필요사항")     # 생성 후 맨 앞으로 이동 (order_sheets가 처리)
ws_q.sheet_properties.tabColor = "FFFF00"      # 시트 인덱스(탭) 색깔: 노란색
ws_q["A1"] = "설치계획서 확인사항"              # A1 고정 문구
# 헤더 (행3) — 4열 고정
ws_q["A3"]="번호"; ws_q["B3"]="구분(시트/단계)"
ws_q["C3"]="확인필요항목 및 내용"; ws_q["D3"]="업체 답변"
# 행4~ : 비워 둔다 (사용자 작성 영역)
```

- 열은 **4열 고정**: `번호` / `구분(시트/단계)` / `확인필요항목 및 내용` / `업체 답변`
- **행4 이하에 아무것도 쓰지 않는다.** 자동 채움 금지 — 사용자가 판단해 기재한다
- 열 너비는 작성 편의를 고려해 넉넉히(예: 6 / 24 / 80 / 40)

## 단위 환산 상수 (항상 적용)

| 환산 | 계수 |
|------|------|
| 1 RT → kW | × 3.517 |
| ※ **시간대별 운전계획(Phase 4)은 예외 — 단위를 환산하지 않고 원문 그대로 쓴다** | — |
| 1 kW → RT | ÷ 3.517 |
| 1 RT → kcal/h | × 3,024 |
| 1 kW → kcal/h | × 860 |
| 1 RTh → kWh | × 3.517 |
| 1 RTh → Mcal | × 3.024 |
| 1 kcal | ÷ 3,024 → RTh |
| 1 m² → 평 | ÷ 3.30579 |
| 물 융해잠열 | 79.68 kcal/kg (333.6 kJ/kg) |

> 모든 단위환산은 **수식을 그대로 엑셀 셀에 기입**한다(상수 곱셈/나눗셈 형태). 값만 적지 않는다.

## 펌프 순환유량 적정성 (LPM/RT) — 항상 검증

각 펌프에 대해 RT당 순환유량을 계산하고 기준(10 LPM/RT)에서 **±10%** 벗어나면
`체크리스트`(행27 판정 / 행28~ 카탈로그 P2)에 포함한다.

```
LPM/RT = 설계유량(LPM) ÷ 담당 냉동능력(RT)
기준범위: 9.0 ~ 11.0 LPM/RT  (= 10 ± 10%)
판정 수식 예: =IF(AND(LPM_RT>=9,LPM_RT<=11),"✅","⚠️ 기준이탈")
```

`펌프` 시트에 펌프별로 `설계유량(LPM)`, `담당능력(RT)`, `LPM/RT(수식)`,
`기준범위(9~11)`, `판정(수식)` 행을 둔다. 수식을 그대로 표시한다.
**담당 냉동능력(RT)의 기준(분모)은 `references/decision_rules.md` §2의 정의를 그대로 사용한다.**

| 규칙 | 내용 |
|------|------|
| 숫자 셀 | 숫자형으로 입력(문자열 금지). 단위는 지정된 단위 열/괄호에만 |
| 숫자 표시형식 | **모든 숫자 셀·숫자 반환 수식 셀에 천단위 쉼표 서식**. 정수 `#,##0` / 소수·수식 `#,##0.0`. 예외는 아래 고정 목록만 |
| 환산 표시값 | 소수 1자리 반올림 (예: 13,068.55평 → 13068.6) |
| 문자열 병기 형식 | `숫자(천단위쉼표) 단위 (환산값 단위)` 예: `2,721,600 kcal/h (900 RT)` |
| 온도/기간 범위 | `A → B ℃`, `HH:00~HH:00 (Nh)`, `YYYY-MM-DD ~ YYYY-MM-DD` |
| 수식 셀 | 지정된 수식 그대로. 수식 지정 셀에 값 입력 금지, 값 지정 셀에 수식 입력 금지 |
| **표시용 문자열은 `=`로 시작 금지** | openpyxl은 `=`로 시작하는 문자열을 **수식으로 저장**한다. 산식 설명·출처 표기처럼 **읽으라고 쓴 문자열**이 `=`로 시작하면 엑셀이 파싱에 실패해 **"제거된 레코드: 수식" 경고와 함께 셀이 사라진다**. `= A ÷ B` → `결과(kW) = A ÷ B`, `=C7 (본 시트)` → `본 시트 C7 (…)` 처럼 앞에 설명어를 붙인다 |
| 판정 셀 | `✅` `⚠️` `❌` 기호만 (E열 등 판정 전용 셀). 수식 판정이 지정된 셀은 수식 |
| 시트 간 참조 | 지정된 교차참조 수식(`='시트명'!셀`)을 그대로 사용 |

### 숫자 표시형식 (R6) — 쉼표 필수, `.#` 서식 금지

값을 바꾸지 않고 **표시형식만** 입힌다(수식 셀도 대상 — 엑셀이 계산한 결과에 서식이 먹는다).
판정 수식(`IF(...,"✅","⚠️")`처럼 문자열을 반환하는 셀)은 제외한다.

| 대상 | 서식 | 예 |
|------|------|-----|
| **kcal·kcal/h 단위 셀** (A열 항목명 또는 C열 단위에 `kcal`) | `#,##0` — **항상 소수점 없이** | `2,721,600` · `12,292,560` |
| 값이 **정수**인 셀 | `#,##0` | `13,069` |
| 값에 **소수가 있는** 셀 | `#,##0.0` | `34,561.6` · `666.5` |
| **수식 셀**(결과를 미리 알 수 없음) | `#,##0.0` | `13,068.6` |
| 용량비·축열율 등 **비율(%)** | `0.0%` 또는 `0.0` | `111.5%` · `47.0` |
| 축열능력 차이율(%) `축열조'!B59` | `0.00` | `0.50` |
| 펌프 LPM/RT `펌프'!B{s+7}` | `0.0` | `10.5` · `8.3` |

> **`#,##0.#` 류의 `.#` 서식은 쓰지 않는다.** 엑셀은 이 서식에서 소수부가 없으면
> `900.`처럼 **소수점만 남겨** 표시한다. 정수는 `#,##0`, 그 외는 `#,##0.0`으로 나눈다.

> 셀 맵은 **각 Phase 참조 문서**가 규정한다. 여기에 지정되지 않은 셀에는 아무것도 쓰지 않는다
> (자유 서술·참고·※주석 금지). 문서에 없는 값은 아래 결측값 규칙으로 채운다.

## 출처(수록 위치) 표기 (v17) — 상호검증의 토대

설치계획서는 같은 수치를 본문 요약표·규격표·설계서·데이터시트에 **중복 기재**한다.
그래서 값마다 **어디서 가져왔는지**를 남기고, 여러 곳에 있으면 **전부** 적는다.

| 대상 | 방법 |
|------|------|
| 모든 데이터 시트 | **마지막 열 = `출처(수록 위치)`**. 행마다 `{파일명} p.{n} ({절 제목})` 기입. 기본값은 그 시트의 주 출처, 다른 곳에서 온 값은 행별로 덮어쓴다 |
| 여러 곳에 기재된 값 | 슬래시로 **전부 나열**: `{파일명} p.4 (조수 5) / {파일명} p.21 (수량 3) — 불일치 T1` |
| 시트 전체의 주 출처 | **B1** = `수록: {파일명} p.{n} ({절 제목})`. 데이터시트가 따로 있으면 ` / 데이터시트 p.{n}` 병기 |
| `건물부하계산서`·`시간대별운전계획` | 이미 `[1]` 블록에 `수록 위치` 행이 있다(중복 기재 금지) |
| `체크리스트` | **F열 = `근거 위치`**. 전 행에 기입. 원문이 없으면 `해당없음` |

- 파일명은 **그대로** 쓴다(축약·번호 치환 금지). 설계서·데이터시트는 접두어로 구분:
  `설계서 {파일명} p.{n}` / `데이터시트 {파일명} p.{n}`.

## `상호검증` 시트 (v17) — 중복 기재 수치 대조

A1 = `설치계획서 내부 상호검증 (중복 기재 수치 대조)`. 행3 헤더 10열 고정:

```
번호 없음 | A=항목 | B=출처① | C=값① | D=출처② | E=값② | F=출처③ | G=값③
         | H=허용오차 | I=일치 판정(수식) | J=관련 코드
```

- **행4~**: 두 곳 이상에 기재된 수치를 한 행씩. 값은 **숫자형**, 출처는 `{파일명} p.{n}`.
- **I열 판정 = 수식 고정**:
  `=IF(COUNT(C{r},E{r},G{r})<2,"단일 출처",IF(MAX(C{r},E{r},G{r})-MIN(C{r},E{r},G{r})<=H{r},"✅","⚠️ 불일치"))`
  (`MAX`/`MIN`은 빈 셀을 무시하므로 출처가 2곳이어도 그대로 동작한다)
- **허용오차는 `decision_rules.md` §5.5 고정표**를 따른다: 연면적·심야전력 설비용량 **0.1%**,
  유량·열량 **1%**, 그 외 **완전일치**. `허용기준` 열에 문구, `허용오차(수식)` 열에 `=C{r}*0.001` 형태 수식.
- **애초에 다른 값끼리는 대조하지 않는다**(예: 축열조 내부체적 vs 총유체량). 대조표 아래 ※ 행에 명시.
- **J열 관련 코드**: 불일치가 교차검증 카탈로그에 해당하면 코드(`K2`·`T1`·`X2` …)를 적는다.
  코드가 없는 불일치가 나오면 **카탈로그에 코드를 신설**하고 나서 기재한다(R4).
- 마지막 데이터 행 +2부터 **[요약]** 블록: 대조 항목 수 / 일치 / 불일치 / 단일 출처 (COUNTIF 수식).

## 체크리스트 행 구성 (v17)

```
행7~27    필수 21개 심사항목            A열 = {n}단계
행28~     교차검증 카탈로그 발견 항목    A열 = 추가
그 다음+1  각 시트 확인사항 통합          A열 = 시트확인
```

**`시트확인` 블록**: 다른 시트의 확인사항([4]·[5] 블록, 검증 블록의 ⚠️/❌ 사유)을 체크리스트로
모아 한곳에서 보게 한다. B열=항목, C열=계획서 기재, D열=검증값, E열=`⚠️`, F열=근거 위치.
`업체확인필요사항`은 빈 양식이므로, **분석 결과는 전부 체크리스트에 모인다.**
